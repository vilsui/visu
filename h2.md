Karvinen 2026: Sudo without password
- Sudoers-tiedostolla voidaan sallia tietylle käyttäjälle sudo-komennon käyttö ilman salasanaa
- Tiedosto luodaan hakemistoon /etc/sudoers.d/ ja siihen kirjoitetaan rivi: kayttaja ALL=(ALL) NOPASSWD: ALL
- Tiedoston oikeellisuus tarkistetaan komennolla: sudo visudo -c -f /etc/sudoers.d/kayttaja
OMana huomiona tässä se että miksei muokata suoraan /etc/sudoers tiedostoa? Virhe mielestäni siinä että erillinen tiedosto turvallisempi, koska virhe siinä ei riko koko sudo-määritelmätiedostoa.

Munroe 2006: xkcd 149 - Sandwich
- Meemi jossa sudo-komento saa toisen osapuolen tekemään mitä vain
- Tiivistää hyvin sudon merkityksen: sudo-käyttäjällä on oikeudet tehdä asioita, joita normaalisti ei voi
Omana huomiona huomina se, että meemi muistuttaa hyvin siitä mitkä mahdollisuudet ja samalla vastuu sudo käyttäjällä on.

Karvinen 2026: Passwordless Sudo with Ansible
- Ansiblea varten tarvitaan tunnus, jolla on sudo-oikeudet ilman salasanaa
- SSH-avainpohjainen kirjautuminen mahdollistaa Ansiblen käytön ilman käyttäjän toimenpiteitä
- Yhdistämällä passwordless sudo ja SSH-avain saadaan täysin automatisoitu hallintaympäristö

Omana huomiona tässä se, että turvallisuuden näkövinkkelistä olisi hyvä rajata sudo oikeuksia tarkasti.

a) Sudoless

Loin visu-käyttäjälle sudoers-tiedoston joka sallii sudon ilman salasanaa.

    sudo visudo -f /etc/sudoers.d/visu
    (kirjoitin: visu ALL=(ALL) NOPASSWD: ALL)

Tarkistin tiedoston sisällön:
    sudo cat /etc/sudoers.d/visu
    visu ALL=(ALL) NOPASSWD: ALL

Testasin että sudo toimii ilman salasanaa:

    sudo echo "Sudo toimii ilman salasanaa!"
    Sudo toimii ilman salasanaa!

b) Antero
Loin Antero-käyttäjän Ansiblella SSH-avaimineen automaattisesti.
Ensin loin SSH-avainparin Anterolle:
    ssh-keygen -t ed######-f ~/.ssh/antero_key
Roolin sisältö:
    cat roles/antero/tasks/main.yml
    - name: Luo kayttaja Antero
      user:
        name: antero
        create_home: true
        shell: /bin/bash
        state: present

    - name: Lisaa SSH-avain Anterolle
      authorized_key:
        user: antero
        key: "{{ lookup('file', '~/.ssh/antero_key.pub') }}"
        state: present

Ajoin playbookin:
    ansible-playbook -i hosts site.yml
    localhost : ok=6 changed=2 unreachable=0 failed=0

Testasin että käyttäjä on olemassa:
    id antero
    uid=1004 (antero) gid=1004 (antero) groups=1004 (antero)

c) Package
Asensin paketit tree ja curl Ansiblella apt-moduulilla.
Roolin sisältö:
    cat roles/packages/tasks/main.yml
    - name: Asenna paketit
      apt:
        name:
          - tree
          - curl
        state: present
        update_cache: true

Ajoin playbookin: ansible-playbook -i hosts site.yml
    PLAY RECAP
    localhost : ok=6 changed=2 unreachable=0 failed=0
Testasin että paketit asentuivat:
    which tree
    /usr/bin/tree
    which curl
    /usr/bin/curl
    tree --version

Tarkistin tiedoston sisällön ja oikeudet:

    sudo cat /home/antero/viesti.txt
    Tämä on Ansiblen kirjoittama tiedosto.
    Toinen rivi tassa.
    Kolmas rivi viimeiseksi.

    sudo ls -la /home/antero/viesti.txt
    -rw-r----- 1 antero antero 72 Apr 9 12:00 /home/antero/viesti.txt

Oikeuksien selitys: 0640 = -rw-r-----
- Omistaja (antero): rw- = voi lukea ja kirjoittaa
- Ryhmä (antero): r-- = voi ainoastaan lukea
- Muut (esim. visu): --- = ei mitään oikeuksia

Todistin tämän ajamalla komennon visu-käyttäjänä:
    cat /home/antero/viesti.txt
    cat: /home/antero/viesti.txt: Permission denied

e) Jotain muuta: lineinfile-moduuli
Käytin lineinfile-moduulia jota ei ole kurssilla käsitelty.
Se varmistaa, että tietty rivi on olemassa tiedostossa
ilman että koko tiedosto korvataan.

Roolin sisältö:
    cat roles/extrat/tasks/main.yml
    - name: Lisaa rivi tiedostoon lineinfile-moduulilla
      lineinfile:
        path: /home/antero/viesti.txt
        line: "Tama rivi on lisatty lineinfile-moduulilla."
        state: present

Ajoin playbookin:
    ansible-playbook -i hosts site.yml
    PLAY RECAP

    Kolmas rivi lopuksi.
    Tama rivi on lisatty lineinfile-moduulilla.

Idempotenttisuus: jos rivi on jo olemassa, Ansible ei tee muutoksia
(ok eikä changed).
