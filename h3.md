x) Tiivistelmat

Apache2-webpalvelin asennetaan automaattisesti Ansiblella käyttäen package-file-service -mallia - Package-file-servicella tarkoitetaan että asennetaan  paketti, muokataan konffaukset, käynnistetään demoni uudelleen. Konfiguraatiomuutokset eivät astu voimaan ellei demonia käynnistetä uudestaan  käytetään Ansiblen handlers-ominaisuutta. Handleri käynnistää Apachen uudelleen vain jos konffaustiedosto on muuttunut.
Omana huomiona: Package-file-service on kaava demonien hallintaan ja sama logiikka toimii muillakin palvelimilla kuten Nginxillä.

Ansible Community Documentation: Handlers
- Handlerit ovat tehtäviä jotka suoritetaan vain kun jokin muu tehtävä tekee muutoksia
- Notify-avainsanalla ilmoitetaan handlerille että se pitää ajaa
- Handlerit ajetaan vasta playbookin lopussa, ei heti notify-kohdassa

Oma huomio: Hyödyllinen esmes kun halutaan käynnistää demoni uudelleen vain jos konfiguraatio muuttui.

a) Apassi

Asensin Apache2:n ja tein tavallisen käyttäjän muokattavan weppisivun.

Asensin Apachen ja suoritin seuraavat komennot tarkistaakseni että asennus onnistunut ja on käytössä:









    sudo apt install apache2 -y
    sudo systemctl status apache2
 

Tämän jälkeen siirryin testaamaan oletussivun:

    curl http://localhost
 

Siirryin luomaan kansiota ja sivun tavalliselle käyttäjälle:
    mkdir -p /home/visu/publicsites
    chmod 755 /home/visu/
    chmod 755 /home/visu/publicsites/
    nano /home/visu/publicsites/index.html

Index.html sisällöksi laitoin:

    <h1>Hei maailma!</h1>
    <p>Tama on Ville Suikin testisivu.</p>
 

HTML sisällön jälkeen loin  Apache virtual host -konfiguraation:

    sudo nano /etc/apache2/sites-available/visu.conf

Visu.conf sisältö:

    <VirtualHost *:80>
        DocumentRoot /home/visu/publicsites
        <Directory /home/visu/publicsites>
            Require all granted
        </Directory>
    </VirtualHost>

 
Otin uuden sivun käyttöön komennoin:
    sudo a2ensite visu.conf
    Enabling site visu.
    sudo a2dissite 000-default.conf
    sudo systemctl restart apache2
 
[SCREENSHOT: a2ensite ja a2dissite]

Curlaamalla sain seuraavan sisällön:

    curl http://localhost
    <h1>Hei maailma!</h1>
    <p>Tama on Ville Suikin testisivu.</p>

 

 b) Nginx

Asensin Nginxin ja vaihdoin sen Apachen tilalle.
Sammutin ensin Apachen: sudo systemctl stop apache2 ja asensin nginx komennolla sudo apt install nginx -y
 

Tarkistin että Nginx on käynnissä komennolla: sudo systemctl status nginx
 

Tämän jälkeen siirryin luomaan Nginx-konfiguraation:
sudo nano /etc/nginx/sites-available/visu

Visu-tiedoston sisältö oli:

    server {
        listen 80 default_server;
        root /home/visu/publicsites;
        index index.html;
    }
 
Otin sivun käyttöön:

    sudo ln -s /etc/nginx/sites-available/visu /etc/nginx/sites-enabled/visu
    sudo rm /etc/nginx/sites-enabled/default



Testasin konfiguraation:
    sudo nginx -t
    nginx: configuration file /etc/nginx/nginx.conf test is successful

Käynnistin Nginxin uudelleen:
   sudo systemctl restart nginx

 
Curlaamalla testasin että sivu näkyy:

    curl http://localhost
    <h1>Hei maailma!</h1>
    <p>Tama on Ville Suikin testisivu.</p>

 

 c) Automoottorix

Automatisoin Nginxin asennuksen Ansiblella.
Loin roolin:
    mkdir -p roles/nginx/tasks
    nano roles/nginx/tasks/main.yml

Main.yml sisältö:

    $ cat roles/nginx/tasks/main.yml
    - name: Asenna nginx
      apt:
        name: nginx
        state: present
        update_cache: true

    - name: Kaynnista nginx
      service:
        name: nginx
        state: started
        enabled: true
 
Site.yml sisältö:

    cat site.yml
    - hosts: all
      become: true
      roles:
        - nginx
 
Ajoin playbookin:

    $ ansible-playbook -i hosts site.yml
    ...
    PLAY RECAP
    localhost : ok=3 changed=0 unreachable=0 failed=0

 
Testasin että Nginx toimii:

    curl http://localhost
    <h1>Hei maailma!</h1>
